Guide

Severity Classification Criteria

Severity위험도조치판단 조건비고
7 없음 유지 ① Action = Deny
② ICMP 전용 서비스 (Ping 등)
③ RITM 등록 + 스케줄 미만료 (유효한 ITS)
④ 정책명에 "controlled" 포함 (멀티 방화벽 통과)
⑤ 정책명·출발지·목적지에 ADMIN 또는 MGMT 포함		 예외 처리 정책 — 삭제·비활성화 대상 아님
1 Critical 즉시 비활성화 ① 출발지·목적지·서비스 모두 Any·All
② 출발지 또는 목적지 Any·All + Hit Count = 0
③ 출발지 또는 목적지 Any·All + Hit Count > 0이나 Last Used 1년 이상 (장기 방치)
④ FTP·TELNET·TFTP·RLOGIN·RSH 등 위험 서비스 단독 포함
⑤ 위험 서비스 + 정상 서비스 혼합 + Hit Count = 0 또는 Last Used 1년 이상
⑥ BOSK 객체 포함
⑦ 정책명에 임시 키워드(temp·이관 등) + RITM 없음 + Hit Count = 0
⑧ 서비스 ALL + Hit Count = 0 또는 Last Used 1년 이상		 비활성화 후 Hit=0 확인 시 삭제 진행
2 High 삭제 필요 ① 상태 = Disabled
② 스케줄 만료 (YYMMDD 날짜가 오늘 이전)
③ Hit=0 + Accept + Server↔User + 등록 후 1년 이상
④ Hit=0 + Accept + Server↔Server + always + 1년 초과		 등록 1년 미만 정책은 제외. S-S 날짜 스케줄은 2년 초과 기준
3 Medium 추가 검토 필요 ① Server→User + 등록 1년 초과 + Hit < 경과연도×100 + Last Used 1년 이상
② 위험 서비스 + 정상 서비스 혼합 + 활성 사용(1년이내)
③ 출발지·목적지 Any·All + 활성 사용(1년이내) + Server→User
④ 서비스 ALL + 활성 사용(1년이내) + Server→User		 ② 위험 서비스 혼합 정책: 정책 전체 삭제 금지 — 위험 서비스(포트)만 제거, 나머지 정상 서비스는 유지
4 Medium 추가 검토 필요 ① Server↔Server + 등록 2년 초과 + Hit < 50
② Server↔Server + 등록일 미상
③ 출발지·목적지 Any·All + 활성 사용(1년이내) + Server↔Server
④ 서비스 ALL + 활성 사용(1년이내) + Server↔Server		 활성 Any/All 정책은 즉시 비활성화 대신 허용 범위 축소 검토 권장
5 Low ITS Ticket 요청 ① Server→User + 등록 1년 이내 + RITM 없음
② Server→User + 등록 1년 초과 + Hit 부족 + Last Used 1년 미만 (최근 사용 중이나 등록 미비)
③ VDI 관련 객체 포함 (VDI_132~135)
④ 서비스 ALL + 활성 사용(1년이내) + Server→User		 정책은 유지하되 ITS Ticket으로 정식 등록 권장
6 Low ITS Ticket 요청 ① AD·DNS·LDAP 등 인증/DNS 서비스 + Hit > 0
② Server↔Server + 2년 이내 + RITM 없음
③ Server↔Server + Hit ≥ 50
④ 인프라(F1·F2·Internal) 또는 관리(MGMT) 객체 포함		 정책 유지가 필요하면 ITS Ticket으로 정식 등록 권장

※ User IP 대역 미설정 시 통신 유형(Server/User) 판단 불가 — Severity 1·2·7은 IP 대역 없이도 판단 가능
