Guide

Severity Classification Criteria

SeverityRisk LevelActionClassification Criteria (NIST SP 800-41 / CIS Controls)Notes
7 None Keep rule ① Action = Deny (by design)
② ICMP-only service (Ping, diagnostic traffic)
③ Valid change ticket registered + schedule not expired
④ Rule name contains "controlled" (multi-firewall traversal policy)
⑤ Customer-designated admin policy object (configurable)		 Compliant rules — not subject to removal or disabling
1 Critical Disable immediately ① Source, destination, AND service all Any/All (violates least-privilege)
② Source or destination Any/All + Hit Count = 0
③ Source or destination Any/All + Hit Count > 0 but Last Used > 1yr (long idle)
④ Insecure/legacy protocol only (FTP, TELNET, TFTP, RLOGIN, RSH)
⑤ Insecure + normal services mixed + unused or Last Used > 1yr
⑥ Customer-designated high-risk object (configurable)
⑦ Temporary keyword in rule name + no change ticket + unused
⑧ Service ALL + unused or Last Used > 1yr		 Disable first; delete after confirming zero traffic
2 High Delete rule ① Status = Disabled
② Schedule expired (date-based schedule in the past)
③ Hit=0 + Accept + Server→User + registered > 1yr ago
④ Hit=0 + Accept + Server↔Server + always-on + > 1yr old		 Rules registered < 1yr are exempt. Server-Server date schedules use 2yr threshold
3 Medium Review required ① Server→User + > 1yr old + Hit < (age×100) + Last Used > 1yr
② Insecure + normal services mixed + actively used (< 1yr)
③ Source/dest Any/All + active use (< 1yr) + Server→User
④ Service ALL + active use (< 1yr) + Server→User		 ② Mixed insecure services: do NOT delete the whole rule — remove only the insecure service/port
4 Medium Review required ① Server↔Server + > 2yr old + Hit < 50
② Server↔Server + creation date unknown
③ Source/dest Any/All + active use (< 1yr) + Server↔Server
④ Service ALL + active use (< 1yr) + Server↔Server		 For active Any/All rules: consider scope reduction rather than immediate disable
5 Low Keep + request change ticket ① Server→User + registered < 1yr + no change ticket
② Server→User + > 1yr old + low hit count + Last Used < 1yr (active but not formally registered)
③ Customer-designated user segment object (configurable)
④ Service ALL + active use (< 1yr) + Server→User		 Rule is operationally needed — register formally via change management process
6 Low Keep + request change ticket ① AD/DNS/LDAP authentication service + Hit > 0
② Server↔Server + < 2yr old + no change ticket
③ Server↔Server + Hit ≥ 50 (actively used)
④ Customer-designated infrastructure or management object (configurable)		 Rule is actively needed — register formally via change management process

※ Without User IP ranges configured, Server/User traffic type cannot be determined — Severity 1, 2, and 7 can be classified without IP ranges
